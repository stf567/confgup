# UVM CSV Assembler — Stage 1

## Цель
Простой CLI-ассемблер, который переводит человеко-читаемый CSV в промежуточное представление (IR). В режиме `--test` выводится IR в читаемом виде для проверки тестов из спецификации УВМ.

## Формат CSV
Первая строка — заголовок, например:
mnemonic,A,B,CONST,OFFSET,COMMENT

Каждая строка — одна инструкция:
- mnemonic — мнемоника, например LOAD_CONST, READ_MEM, WRITE_MEM, POPCT...
- A, B, CONST, OFFSET — числовые операнды (десятичные или 0xHEX)
- COMMENT — необязательно

Пример:
LOAD_CONST,39,306,39,,; загрузка константы

## Запуск
- Тестовый режим (печать IR): `python assembler.py input.csv out.bin --test`
- Обычный режим (сохранить бинарник): `python assembler.py input.csv out.bin`

## Что выводится в тестовом режиме
Печатается список инструкций с полями: mnemonic, opcode, size, fields (A, B, CONST, OFFSET).
Это позволяет сравнить с тестами из спецификации и убедиться, что поля и значения совпадают.

## Дальнейшие шаги для полного этапа
- Дописать полную таблицу INSTRUCTION_SET, соответствующую вашей спецификации УВМ.
- Реализовать точную упаковку полей в бинарный формат (для генерации корректных команд).
- Добавить валидацию диапазонов битовых полей (например, A: 0..255 если 8 бит).
- Реализовать labels/константы/символьные имена (если требуется).
